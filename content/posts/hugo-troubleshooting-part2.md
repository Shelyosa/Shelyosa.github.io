## 八、部署上线

### 8.1 费用说明
**个人博客零成本即可上线**，主流静态网站托管平台均提供充足免费额度：
- GitHub Pages / Gitee Pages：完全免费，合理使用范围内无流量限制
- Vercel / Netlify：免费版每月 100GB 流量，个人博客完全够用
- 阿里云 OSS / 腾讯云 COS：每月有免费流量额度，超出后按量付费，成本极低

仅可选额外成本：自定义域名，每年约 30~80 元；使用平台自带的二级域名则完全免费。

### 8.2 安全性说明
Hugo 生成的纯静态网站天然具备高安全性：
- 无后端服务、无数据库，免疫 SQL 注入、命令执行、后台漏洞等常见攻击
- 仅提供静态文件读取，服务端无法执行代码，攻击面极小

**需要注意的安全点**：
1. 接入第三方评论系统时，优先选择官方维护的成熟方案，避免引入 XSS 注入风险
2. 使用自定义域名时，务必开启 HTTPS（主流托管平台默认免费提供证书）
3. 不要在公开仓库里提交密钥、Token、隐私联系方式等敏感信息
4. 不要发布侵权、违规内容，避免平台封号与合规风险

### 8.3 主流托管方案对比
| 方案 | 免费额度 | 国内访问速度 | 自动部署 | 推荐指数 | 适合人群 |
|------|----------|--------------|----------|----------|----------|
| Vercel | 100GB/月流量 | 全球加速，国内尚可 | Git 推送自动更新 | ⭐⭐⭐⭐⭐ | 新手首选，省心省力 |
| GitHub Pages | 无限流量（合理使用） | 国内偏慢 | Git 推送自动更新 | ⭐⭐⭐⭐ | 面向海外读者、有 Git 基础 |
| Gitee Pages | 免费 | 国内速度快 | 免费版需手动更新 | ⭐⭐⭐ | 纯国内读者、不想折腾 |
| 阿里云 OSS | 40GB 免费流量 | 国内快 | 需手动上传 | ⭐⭐⭐ | 有云服务使用基础 |

### 8.4 新手首选：Vercel 自动部署
无需购买服务器，推送代码自动更新，是目前最省心的方案。

#### 前置准备
1. 注册 GitHub 账号，将本地博客项目推送到 GitHub 公开仓库
2. 注册 Vercel 账号，直接使用 GitHub 账号登录即可

#### 部署步骤
1. 打开 Vercel 控制台，点击「Add New」→「Project」
2. 导入你刚才创建的 GitHub 博客仓库
3. 配置项全部保持默认，框架会自动识别为 Hugo
4. 点击「Deploy」，等待 1~2 分钟即可部署完成

#### 后续更新
以后写完新文章，只需要把代码推送到 GitHub，Vercel 会自动重新构建部署，全程无需手动操作。

### 8.5 通用手动部署方法
所有静态托管平台通用，适合任何场景：
1. 在项目根目录的 PowerShell 中执行打包命令：
```powershell
hugo
```
2. 执行完成后，项目根目录会生成 `public` 文件夹，里面就是全部可部署的静态网页文件
3. 把 `public` 文件夹内的所有文件，上传到托管平台的网站根目录即可

## 九、图片管理

### 9.1 本地图片存放规范
所有本地图片统一放在 `static/` 目录下，Hugo 构建时会自动把 static 里的文件原样复制到网站根目录。

推荐目录结构：
```
myblog/
└── static/
    └── images/
        ├── posts/      # 文章配图，可按文章分子文件夹
        │   └── hugo-guide/
        └── avatar.jpg  # 头像、封面等公用图片
```

### 9.2 正确引用方式
在 Markdown 文章中，路径直接从网站根目录写起，不需要加 `static`：
```markdown
![图片描述文字](/images/posts/hugo-guide/cover.jpg)
```

**注意**：路径开头的 `/` 不能省略，代表网站根目录，否则文章子页面会出现路径加载错误。

### 9.3 图片优化建议
1. **控制文件大小**：单张正文图建议压缩到 200KB 以内，封面图不超过 500KB
2. **格式选择**：照片用 WebP/JPG，截图用 PNG，图标用 SVG
3. **尺寸适配**：文章内图宽度不超过 1200px，避免浪费加载流量
4. **命名规范**：全部使用英文小写+数字，不要用中文文件名，避免部分平台加载失败

### 9.4 进阶：图床方案
如果图片数量很多、担心占用仓库空间，可以使用第三方图床托管：
- 免费方案：GitHub 仓库作为图床 + jsDelivr 免费 CDN 加速
- 付费方案：阿里云 OSS、腾讯云 COS、七牛云，按量付费成本极低
使用时直接把图片链接粘贴到文章里即可，和本地引用显示效果完全一致。

## 十、评论系统配置

### 10.1 推荐方案：Giscus
PaperMod 原生适配 Giscus，基于 GitHub Discussions，完全免费、无广告、加载速度快，非常适合技术类博客。

#### 前置条件
1. 拥有 GitHub 账号
2. 你的博客代码仓库是公开仓库
3. 仓库已开启 Discussions 功能（Settings → Features → 勾选 Discussions）

#### 配置步骤
1. 打开 [Giscus 官网](https://giscus.app/zh-CN)
2. 按照页面指引，输入你的 GitHub 仓库名，完成配置后会生成一段嵌入代码
3. 在你的博客项目中新建文件：`layouts/partials/comments.html`
4. 把 Giscus 生成的代码完整粘贴进该文件
5. 在 `hugo.toml` 的 `[params]` 下添加全局开关：
```toml
comments = true
```

#### 效果说明
所有文章底部会自动出现评论区，访客使用 GitHub 账号登录即可评论，评论数据存储在你自己仓库的 Discussions 中，数据完全可控。

### 10.2 其他备选方案
- **Utterances**：同样基于 GitHub Issues，更轻量，功能简洁
- **Waline**：开源评论系统，需要搭配后端服务，支持匿名评论
- **Disqus**：国外老牌评论系统，国内访问慢、有广告，不推荐
- 国内第三方评论系统大多已停止维护，不建议使用

### 10.3 单篇文章单独关闭评论
如果某篇文章不想开启评论，在文章头部配置里添加：
```yaml
comments: false
```
