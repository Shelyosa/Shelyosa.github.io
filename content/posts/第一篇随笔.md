## 一、学习前言

最近从零开始学习使用 Hugo 搭建个人静态博客网站。

Hugo 是目前速度最快的静态网站生成器，无需服务器、无需数据库，轻量化、简洁，适合用来搭建个人博客、学习笔记、技术文档网站。

本次搭建全程使用 Windows 本地环境，从零配置，完整记录搭建流程，方便以后复盘学习。

---

## 二、搭建环境

| 项目 | 配置 |
|------|------|
| 操作系统 | Windows 10 / Windows 11 |
| 搭建工具 | Hugo Extended（最新版） |
| 博客主题 | PaperMod（极简清爽开源主题） |
| 部署平台 | GitHub Pages |
| 运行方式 | 本地预览 + 静态文件打包部署 |

## 三、完整搭建步骤

### 1. 安装 Hugo

从 [Hugo 官网](https://gohugo.io/installation/) 下载 **Hugo Extended** 扩展版（支持 Sass/SCSS 等高级功能），解压后将 `hugo.exe` 所在目录添加到系统环境变量 `PATH` 中。

终端验证安装：

```powershell
hugo version
```

输出类似 `hugo v0.163.0+extended windows/amd64` 即表示安装成功。

### 2. 创建博客站点

```powershell
hugo new site myblog
cd myblog
```

这会生成标准 Hugo 项目结构：

```
myblog/
├── archetypes/      # 内容模板
├── content/         # 文章目录
├── data/            # 数据文件
├── layouts/         # 自定义布局
├── static/          # 静态资源（图片、CSS）
├── themes/          # 主题目录
└── hugo.toml        # 站点配置文件
```

### 3. 安装 PaperMod 主题

使用 Git 子模块方式安装（推荐，方便后续更新）：

```powershell
git init
git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
```

### 4. 配置站点

编辑 `hugo.toml`，添加主题和基础配置：

```toml
baseURL = "https://你的用户名.github.io/"
languageCode = "zh-cn"
title = "我的Hugo博客"
theme = "PaperMod"

[params]
  defaultTheme = "auto"    # 自动跟随系统明暗模式
  ShowReadingTime = true   # 显示阅读时间
  ShowWordCount = true     # 显示字数统计
  ShowToc = true           # 显示文章目录
  TocOpen = true           # 目录默认展开

[[menu.main]]
  name = "首页"
  url = "/"
  weight = 1

[[menu.main]]
  name = "文章"
  url = "/posts/"
  weight = 2

[[menu.main]]
  name = "归档"
  url = "/archives/"
  weight = 3

[[menu.main]]
  name = "标签"
  url = "/tags/"
  weight = 4

[[menu.main]]
  name = "关于"
  url = "/about/"
  weight = 5
```

### 5. 创建第一篇文章

```powershell
hugo new posts/hello-world.md
```

编辑生成的文件，填写 front matter 和正文：

```markdown
+++
title = "我的第一篇文章"
date = "2026-06-15"
tags = ["随笔"]
categories = ["生活"]
description = "这是我的第一篇 Hugo 博客文章。"
draft = false
+++

## 你好，世界！

这是我的第一篇 Hugo 博客文章，记录一下搭建过程。
```

### 6. 本地预览

```powershell
hugo server
```

打开浏览器访问 `http://localhost:1313` 即可实时预览，修改文件后自动刷新。

### 7. 自定义样式

在 `assets/css/extended/` 目录下创建 `custom.css`，可以覆盖主题默认样式：

```css
:root {
    --main-width: 960px;
    --nav-width: 1380px;
}
```

> **重要**：不要直接修改 `themes/PaperMod/` 下的文件！主题目录是 Git 子模块，修改无法同步到远程仓库。所有自定义都应放在项目根目录的 `assets/`、`layouts/`、`static/` 下。

### 8. 构建静态文件

```powershell
hugo
```

生成的静态文件会输出到 `public/` 目录，可以直接部署到任何静态托管平台。

### 9. 部署到 GitHub Pages

**方式一：直接推送 public 目录**

```powershell
cd public
git init
git add .
git commit -m "deploy"
git remote add origin https://github.com/你的用户名/你的用户名.github.io.git
git push -u origin main
```

**方式二：使用 GitHub Actions 自动部署**

在仓库根目录创建 `.github/workflows/hugo.yml`：

```yaml
name: Deploy Hugo to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'
          extended: true
      - name: Build
        run: hugo --minify
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

推送代码后，GitHub Actions 会自动构建并部署，几分钟后可在 `https://你的用户名.github.io/` 访问。

## 四、踩坑记录

### 坑1：主题文件修改无法同步

直接修改 `themes/PaperMod/` 下的文件，本地预览正常，但部署到 GitHub Pages 后样式丢失。

**原因**：主题目录是 Git 子模块，对子模块的修改不会自动提交到父仓库。

**解决**：通过 `assets/css/extended/custom.css` 覆盖样式，不直接修改主题文件。

### 坑2：本地与线上宽度不一致

本地 `hugo server` 预览文章宽度正常（960px），但部署后变窄（720px）。

**原因**：之前在主题的 `theme-vars.css` 中修改了 `--main-width`，但子模块修改未同步。

**解决**：在 `custom.css` 中重新声明 `--main-width: 960px`。

### 坑3：新页面 404

新建页面后访问报 404，Hugo 服务崩溃。

**原因**：front matter 中 `layout` 字段值写错了，且 Hugo 热更新在解析失败后连续触发导致 panic。

**解决**：修正 front matter 语法，重启 `hugo server`。

## 六、总结

Hugo 的上手门槛很低，从安装到部署大约半天时间就能跑通全流程。核心要点：

1. **用 Extended 版本**，支持 Sass 等高级功能
2. **主题用子模块安装**，方便更新
3. **自定义放项目根目录**，不要改主题文件
4. **GitHub Actions 自动部署**，省去手动推送 public 目录的麻烦
5. **本地预览是开发利器**，所有修改先在本地验证再推送
